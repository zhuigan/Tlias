# 前后端交互详解

## 📌 项目类型说明

这是一个**非前后端分离**的项目，也叫**单体应用**或**传统Web应用**。

### 特点
- ✅ 前端和后端在同一个项目中
- ✅ 部署在同一个服务器（Tomcat）
- ✅ 使用同一个端口（8080）
- ✅ 不存在跨域问题
- ❌ 前端不是独立的 Vue/React 项目

## 🔄 完整交互流程

### 流程1：用户登录

#### 前端代码（index.html）
```javascript
// 1. 用户点击登录按钮，触发 login 函数
async function login(event) {
    event.preventDefault();
    const username = document.getElementById('username').value;  // 获取用户名
    const password = document.getElementById('password').value;  // 获取密码

    // 2. 使用 fetch API 发送 HTTP POST 请求
    const response = await fetch('http://localhost:8080/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password })  // 发送 JSON 数据
    });
    
    // 3. 接收后端返回的 JSON 响应
    const result = await response.json();
    
    // 4. 根据返回结果更新页面
    if (result.code === 1) {
        token = result.data;  // 保存 token
        showMessage('loginMessage', '登录成功！', 'success');
    }
}
```

#### 后端代码（LoginController.java）
```java
@RestController
@RequestMapping("/login")
public class LoginController {
    
    @Autowired
    private EmpService empService;

    // 1. 接收前端的 POST 请求
    @PostMapping
    public Result login(@RequestBody Emp emp) {
        // 2. 从请求体中获取用户名和密码
        // @RequestBody 自动将 JSON 转换为 Emp 对象
        
        // 3. 调用 Service 层验证用户
        Emp empData = empService.getByUsernameAndPassword(
            emp.getUsername(), 
            emp.getPassword()
        );
        
        // 4. 验证失败，返回错误信息
        if (empData == null) {
            return Result.error("用户名或者密码错误");
        }

        // 5. 验证成功，生成 JWT Token
        Map<String, Object> map = new HashMap<>();
        map.put("id", empData.getId());
        map.put("username", empData.getUsername());
        String token = JwtUtils.generateJwt(map);

        // 6. 返回 JSON 响应（Spring Boot 自动转换为 JSON）
        return Result.success(token);
    }
}
```

#### 数据流转
```
前端发送:
POST http://localhost:8080/login
Content-Type: application/json
{
  "username": "admin",
  "password": "123456"
}

后端返回:
HTTP/1.1 200 OK
Content-Type: application/json
{
  "code": 1,
  "msg": "success",
  "data": "eyJhbGciOiJIUzI1NiJ9..."  ← JWT Token
}
```

### 流程2：查询部门列表

#### 前端代码
```javascript
async function loadDepts() {
    // 1. 发送 GET 请求，携带 token
    const response = await fetch('http://localhost:8080/depts', {
        headers: { 'token': token }  // 在请求头中携带 token
    });
    
    // 2. 接收 JSON 响应
    const result = await response.json();

    // 3. 解析数据并渲染到表格
    if (result.code === 1) {
        const tbody = document.querySelector('#deptTable tbody');
        tbody.innerHTML = result.data.map(dept => `
            <tr>
                <td>${dept.id}</td>
                <td>${dept.name}</td>
                <td>${dept.createTime}</td>
                <td>${dept.updateTime}</td>
            </tr>
        `).join('');
    }
}
```

#### 后端代码
```java
@RestController
@RequestMapping("/depts")
public class DeptController {
    
    @Autowired
    private DeptService deptService;

    // 1. 接收 GET 请求
    @GetMapping
    public Result list() {
        // 2. 调用 Service 查询数据库
        List<Dept> depts = deptService.list();
        
        // 3. 返回 JSON 响应
        return Result.success(depts);
    }
}
```

#### 拦截器验证 Token
```java
@Component
public class LoginCheckInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) {
        // 1. 从请求头获取 token
        String token = request.getHeader("token");
        
        // 2. 验证 token
        if (token == null || token.isEmpty()) {
            // 未登录，返回错误
            Result notLogin = Result.error("NOT_LOGIN");
            response.getWriter().write(JSON.toJSONString(notLogin));
            return false;  // 拦截请求
        }
        
        // 3. 校验 token 是否有效
        try {
            JwtUtils.parseJWT(token);
        } catch (Exception e) {
            // token 无效
            return false;
        }
        
        // 4. 放行请求
        return true;
    }
}
```

#### 数据流转
```
前端发送:
GET http://localhost:8080/depts
token: eyJhbGciOiJIUzI1NiJ9...

↓ 经过拦截器验证 token

后端返回:
{
  "code": 1,
  "msg": "success",
  "data": [
    {"id": 1, "name": "学工部", "createTime": "2023-09-01", ...},
    {"id": 2, "name": "教研部", "createTime": "2023-09-01", ...}
  ]
}
```

## 🎯 关键技术点

### 1. Spring Boot 自动配置

```java
@SpringBootApplication  // 启动类注解
public class TliasWebManagementApplication {
    public static void main(String[] args) {
        SpringApplication.run(TliasWebManagementApplication.class, args);
        // Spring Boot 自动：
        // - 启动内置 Tomcat
        // - 扫描 Controller
        // - 配置静态资源路径
        // - 初始化数据库连接
    }
}
```

### 2. 静态资源映射

Spring Boot 默认静态资源路径：
```
/static/
/public/
/resources/
/META-INF/resources/
```

访问规则：
```
文件位置: src/main/resources/static/index.html
访问URL:  http://localhost:8080/index.html
```

### 3. RESTful API 设计

```java
@RestController  // = @Controller + @ResponseBody
@RequestMapping("/depts")
public class DeptController {
    
    @GetMapping              // GET /depts - 查询所有
    @GetMapping("/{id}")     // GET /depts/1 - 查询单个
    @PostMapping             // POST /depts - 添加
    @PutMapping              // PUT /depts - 修改
    @DeleteMapping("/{id}")  // DELETE /depts/1 - 删除
}
```

### 4. JSON 自动转换

Spring Boot 使用 Jackson 自动转换：

```java
// 后端返回对象
return Result.success(depts);

// 自动转换为 JSON
{
  "code": 1,
  "msg": "success",
  "data": [...]
}
```

```java
// 前端发送 JSON
body: JSON.stringify({ username, password })

// 后端自动转换为对象
public Result login(@RequestBody Emp emp) {
    // emp.getUsername() 可以直接使用
}
```

### 5. 跨域处理

因为前后端在同一个域（localhost:8080），所以：
- ✅ 不存在跨域问题
- ✅ 不需要 CORS 配置
- ✅ 可以直接发送请求

如果前后端分离（不同端口），需要配置：
```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins("http://localhost:3000")
                .allowedMethods("GET", "POST", "PUT", "DELETE");
    }
}
```

## 📊 完整请求响应周期

```
1. 用户操作
   ↓
2. 浏览器执行 JavaScript
   ↓
3. fetch() 发送 HTTP 请求
   ↓
4. Tomcat 接收请求
   ↓
5. 拦截器验证 token（如果需要）
   ↓
6. DispatcherServlet 路由到 Controller
   ↓
7. Controller 调用 Service
   ↓
8. Service 调用 Mapper
   ↓
9. Mapper 执行 SQL 查询数据库
   ↓
10. 数据层层返回
   ↓
11. Controller 返回 Result 对象
   ↓
12. Jackson 将对象转换为 JSON
   ↓
13. Tomcat 发送 HTTP 响应
   ↓
14. JavaScript 接收 JSON 数据
   ↓
15. 更新页面 DOM
   ↓
16. 用户看到结果
```

## 🆚 对比：前后端分离 vs 非分离

### 非前后端分离（当前项目）

```
优点:
✅ 部署简单（一个 jar 包）
✅ 不存在跨域问题
✅ 开发配置简单
✅ 适合小型项目

缺点:
❌ 前端技术受限（不能用 Vue/React 脚手架）
❌ 前后端耦合
❌ 不利于团队协作
❌ 不利于前端工程化
```

### 前后端分离

```
优点:
✅ 前后端独立开发
✅ 可以使用现代前端框架
✅ 前端工程化（Webpack、Vite）
✅ 利于团队协作

缺点:
❌ 需要处理跨域
❌ 部署复杂（需要两个服务器）
❌ 开发配置复杂
❌ 需要 Nginx 等反向代理
```

## 📚 总结

当前项目的前后端联系方式：

1. **物理位置**：前端 HTML 和后端 Java 代码在同一个项目中
2. **运行方式**：同一个 Tomcat 服务器，同一个端口
3. **通信方式**：JavaScript 通过 fetch() 调用后端 API
4. **数据格式**：JSON（前后端都使用 JSON 交换数据）
5. **认证方式**：JWT Token（存储在前端变量中，请求时携带）

这种方式简单直接，适合学习和小型项目！

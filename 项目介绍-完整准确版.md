# Tlias 员工管理系统 - 项目介绍

## 📋 项目概述

tlias-web-management 是基于 Spring Boot + MyBatis 构建的**单体 Web 应用**，实现了完整的员工与部门管理功能。系统采用传统的 MVC 架构，前后端部署在同一服务器，通过 RESTful API 进行数据交互。

### 核心特性
- 🏢 部门管理（增删改查）
- 👥 员工管理（增删改查、分页查询、条件筛选）
- 🔐 用户认证（登录、注册、JWT 无状态认证）
- 👤 角色权限（管理员/普通用户权限区分）
- 📸 文件上传（阿里云 OSS + 本地存储双重方案）
- 🎨 响应式前端界面（原生 HTML/CSS/JavaScript）

## 🏗️ 系统架构

### 架构模式
**非前后端分离架构**（单体应用）
- 前端静态资源和后端代码在同一项目中
- 部署在同一个 Tomcat 服务器
- 使用同一个端口（8080）
- 不存在跨域问题

### 分层架构
```
表现层 (Controller)  ← RESTful API 接口
业务逻辑层 (Service)  ← 业务处理
数据访问层 (Mapper)   ← MyBatis ORM
模型层 (POJO)         ← 实体类
```

## 🔑 核心功能

### 1. 用户认证与授权
- 用户登录（用户名 + 密码）
- 用户注册（新员工自主注册）
- JWT Token 认证（有效期 7 天）
- 角色权限控制（管理员/普通用户）

### 2. 部门管理
- 查询所有部门
- 添加/修改/删除部门
- 根据 ID 查询部门详情

### 3. 员工管理
- 分页查询员工列表
- 多条件筛选（姓名、性别、入职日期）
- 添加/修改/删除员工
- 批量删除员工
- 按部门查询（普通用户只能查看同部门）

### 4. 文件上传
- 优先上传到阿里云 OSS
- OSS 失败自动切换到本地存储
- 支持图片格式，最大 10MB
- 头像上传和更新功能

### 5. 权限控制
- **管理员** (role = 1)：
  - 访问管理后台
  - 查看所有数据
  - 完整的增删改查权限
  
- **普通用户** (role = 2)：
  - 访问个人中心
  - 只能查看自己和同部门同事
  - 只能上传自己的头像

## 🗄️ 数据库设计

### 部门表 (dept)
- id, name, create_time, update_time

### 员工表 (emp)
- id, username, password, name, gender
- image, job, entrydate, dept_id
- **role** (1:管理员, 2:普通用户)
- create_time, update_time

## 🔌 API 接口

### 认证接口
- POST /login - 用户登录
- POST /register - 用户注册

### 部门接口
- GET /depts - 查询所有部门
- POST /depts - 添加部门
- PUT /depts - 修改部门
- DELETE /depts/{id} - 删除部门

### 员工接口
- GET /emps - 分页查询（支持条件筛选）
- GET /emps/current - 获取当前用户
- POST /emps - 添加员工
- PUT /emps - 修改员工
- PUT /emps/avatar - 更新头像
- DELETE /emps/{ids} - 批量删除

### 文件接口
- POST /upload - 上传文件

## 📊 技术实现

### 分页查询
- 使用 PageHelper 插件实现物理分页
- 支持按姓名、性别、入职日期范围筛选
- 通过 PageBean 封装分页结果

### 数据库处理
- 连接本地或远程 MySQL 数据库
- 使用 MyBatis 作为 ORM 框架
- 支持驼峰命名自动映射
- 动态 SQL 实现条件查询

### 安全机制
- JWT 无状态认证
- 登录拦截器保护所有接口
- 基于角色的访问控制（RBAC）
- Token 存储在前端 localStorage

### 统一响应格式
```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

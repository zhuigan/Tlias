# 权限管理与头像上传功能说明

## 📋 功能概述

系统现在支持：
1. 用户角色区分（管理员 / 普通用户）
2. 头像上传到阿里云OSS
3. 基于角色的权限控制
4. 不同角色看到不同的页面

## 👥 用户角色

### 1. 管理员 (role = 1)
- 用户名: `admin`
- 密码: `123456`
- 权限:
  - 查看所有部门和员工
  - 添加、编辑、删除部门
  - 添加、编辑、删除员工
  - 上传和修改所有员工头像
  - 访问管理后台 (`/admin.html`)

### 2. 普通用户 (role = 2)
- 测试账号: `zhangwuji` / `123456`
- 权限:
  - 只能查看自己的信息
  - 只能查看同部门同事的部分信息
  - 只能上传和修改自己的头像
  - 访问个人中心 (`/user.html`)

## 🔐 登录流程

1. 访问 `http://localhost:8080/index.html`
2. 输入用户名和密码
3. 系统自动识别角色:
   - 管理员 → 跳转到 `/admin.html`
   - 普通用户 → 跳转到 `/user.html`

## 📸 头像上传功能

### 管理员上传头像

在 `/admin.html` 页面:
1. 点击"添加员工"或"编辑员工"
2. 在表单中点击"选择图片"按钮
3. 选择图片文件（支持 jpg, png, gif 等）
4. 图片自动上传到阿里云OSS
5. 保存员工信息时，头像URL一起保存

### 普通用户上传头像

在 `/user.html` 页面:
1. 点击头像右下角的相机图标
2. 选择图片文件
3. 图片自动上传并更新

### 上传限制
- 文件类型: 仅支持图片格式
- 文件大小: 最大 5MB
- 存储位置: 阿里云OSS

## 🗄️ 数据库变更

### 新增字段

在 `emp` 表中添加了 `role` 字段:

```sql
ALTER TABLE emp ADD COLUMN role TINYINT(4) DEFAULT 2 COMMENT '角色, 1:管理员, 2:普通用户';
```

### 更新现有数据

```sql
-- 将 admin 用户设置为管理员
UPDATE emp SET role = 1 WHERE username = 'admin';

-- 其他用户默认为普通用户
UPDATE emp SET role = 2 WHERE username != 'admin';
```

## 🔧 后端接口变更

### 1. 登录接口 (`POST /login`)

返回的 JWT Token 中包含:
- `id`: 用户ID
- `username`: 用户名
- `name`: 姓名
- `role`: 角色 (1=管理员, 2=普通用户)
- `deptId`: 部门ID

### 2. 员工列表接口 (`GET /emps`)

- 管理员: 返回所有员工
- 普通用户: 只返回同部门员工

### 3. 当前用户接口 (`GET /emps/current`)

返回当前登录用户的完整信息

### 4. 头像更新接口 (`PUT /emps/avatar`)

- 普通用户只能更新自己的头像
- 管理员可以通过编辑员工功能更新任何人的头像

### 5. 文件上传接口 (`POST /upload`)

上传图片到阿里云OSS，返回图片URL

## ⚠️ 注意事项

### 1. 阿里云OSS配置

确保 `AliOSSUtils.java` 中的配置正确:
- `endpoint`: OSS地域节点
- `accessKeyId`: 访问密钥ID
- `accessKeySecret`: 访问密钥Secret
- `bucketName`: 存储桶名称

### 2. 跨域问题

如果前端单独部署，需要在 `WebConfig.java` 中配置CORS

### 3. Token过期

JWT Token 默认有效期为7天，过期后需要重新登录

### 4. 图片访问

上传到阿里云OSS的图片需要设置为公共读权限，否则无法在网页中显示

## 🔍 故障排查

### 问题1: 登录后没有跳转

检查浏览器控制台是否有错误，确认token解析是否正确

### 问题2: 图片上传失败

1. 检查阿里云OSS配置是否正确
2. 检查网络连接
3. 查看后端控制台日志

### 问题3: 普通用户看不到同事

1. 确认用户有 `deptId`
2. 确认同部门有其他员工
3. 检查后端接口是否正确过滤

### 问题4: 头像不显示

1. 检查图片URL是否正确
2. 确认OSS存储桶权限设置为公共读
3. 检查浏览器控制台网络请求

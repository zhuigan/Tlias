# 操作日志功能 - 快速启动指南

## 🚀 5分钟快速部署

### 步骤 1：创建数据库表（1分钟）

```bash
mysql -u root -p tlias < database/operation_log.sql
```

输入密码后回车，看到成功提示即可。

### 步骤 2：重启后端（2分钟）

```bash
# 停止当前运行的后端（如果有）
Ctrl+C

# 重新启动
mvn spring-boot:run
```

等待看到 "Started TliasWebManagementApplication" 提示。

### 步骤 3：启动前端（1分钟）

```bash
cd tlias-vue-frontend
npm run dev
```

### 步骤 4：测试功能（1分钟）

1. 打开浏览器访问：http://localhost:5173
2. 使用管理员账号登录
3. 点击左侧菜单"操作日志"
4. 看到日志记录即成功！

## 🎯 一键部署（推荐）

### Windows

双击运行：
```
setup-operation-log.bat
```

### Linux/Mac

```bash
chmod +x setup-operation-log.sh
./setup-operation-log.sh
```

按提示输入数据库信息，脚本会自动完成所有配置。

## 📝 验证部署

### 1. 检查数据库

```sql
USE tlias;
SHOW TABLES LIKE 'operation_log';
```

应该看到 `operation_log` 表。

### 2. 检查后端日志

后端控制台应该显示：
```
📝 操作日志: admin - 用户登录 - 成功
```

### 3. 检查前端页面

访问"操作日志"菜单，应该看到：
- 统计卡片（总数、今日、成功、失败）
- 日志列表
- 自动刷新开关

## 🔧 常见问题

### Q1: 看不到"操作日志"菜单？

**A**: 确保使用管理员账号登录。普通用户无权访问。

### Q2: 日志列表是空的？

**A**: 
1. 先执行一些操作（登录、注册等）
2. 点击刷新按钮
3. 检查后端控制台是否有日志输出

### Q3: 提示"表不存在"？

**A**: 执行数据库脚本：
```bash
mysql -u root -p tlias < database/operation_log.sql
```

### Q4: 自动刷新不工作？

**A**: 
1. 检查开关是否开启
2. 刷新页面重试
3. 查看浏览器控制台是否有错误

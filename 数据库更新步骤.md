# 数据库更新步骤

## 问题说明

错误信息：`Unknown column 'role' in 'field list'`

这是因为数据库中的 `emp` 表还没有 `role` 字段。

## 解决方法

### 方法1: 使用MySQL命令行

1. 打开命令行，登录MySQL：
```bash
mysql -u root -p
# 输入密码: 123456
```

2. 执行以下SQL命令：
```sql
USE tlias;

ALTER TABLE emp ADD COLUMN role TINYINT(4) DEFAULT 2 COMMENT 'role 1=admin 2=user';

UPDATE emp SET role = 1 WHERE username = 'admin';

UPDATE emp SET role = 2 WHERE username != 'admin';

SELECT id, username, name, role FROM emp;
```

3. 看到结果后，输入 `exit` 退出

### 方法2: 使用Navicat或其他数据库工具

1. 连接到 MySQL 数据库
2. 选择 `tlias` 数据库
3. 打开查询窗口
4. 复制粘贴以下SQL并执行：

```sql
-- 添加 role 字段
ALTER TABLE emp ADD COLUMN role TINYINT(4) DEFAULT 2 COMMENT 'role 1=admin 2=user';

-- 设置管理员
UPDATE emp SET role = 1 WHERE username = 'admin';

-- 设置普通用户
UPDATE emp SET role = 2 WHERE username != 'admin';

-- 验证结果
SELECT id, username, name, role, dept_id FROM emp;
```

### 方法3: 重新导入完整的数据库

如果上面的方法不行，可以重新导入数据库：

1. 删除现有数据库：
```sql
DROP DATABASE IF EXISTS tlias;
```

2. 重新导入：
```bash
mysql -u root -p < database/tlias.sql
# 输入密码: 123456
```

## 验证是否成功

执行以下SQL查看 role 字段是否存在：

```sql
USE tlias;
DESCRIBE emp;
```

应该能看到 `role` 字段，类型为 `tinyint(4)`

或者查询数据：

```sql
SELECT id, username, name, role FROM emp WHERE username = 'admin';
```

应该显示 admin 的 role 为 1

## 完成后

1. 重启后端服务
2. 重新登录测试
